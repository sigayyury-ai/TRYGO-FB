# Правила проверки логов

## Когда пользователь просит проверить логи

1. **СНАЧАЛА проверяй ошибки, которые пользователь показывает:**
   - Ошибки компиляции (Vite, TypeScript, React)
   - Ошибки рантайма в браузере
   - Конкретные файлы и строки, указанные в ошибке

2. **Не переключайся на другие логи без явного указания:**
   - Если пользователь показывает ошибку компиляции → проверяй файл и строки
   - Если пользователь показывает ошибку бэкенда → проверяй бэкенд логи
   - Не начинай искать в других местах, пока не исправил показанную проблему

3. **При получении конкретной ошибки:**
   - Сразу открывай указанный файл
   - Находи проблему на указанных строках
   - Исправляй, а не ищи логи в терминале/других местах

## Примеры

**Неправильно:**
```
Пользователь: "читай логи" + показывает ошибку компиляции в MonthlySchedule.tsx:124
AI: начинает проверять логи бэкенда в /tmp/trygo-backend.log
```

**Правильно:**
```
Пользователь: "читай логи" + показывает ошибку компиляции в MonthlySchedule.tsx:124
AI: открывает MonthlySchedule.tsx, проверяет строку 124, находит проблему, исправляет
```
